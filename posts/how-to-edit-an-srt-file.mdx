## What Is an SRT File and Why Would You Need to Edit One?

An SRT (SubRip Subtitle) file is a plain-text file that contains subtitle text paired with timestamps. It's the most widely supported subtitle format across video platforms, media players, and editing tools.

Common reasons people need to edit SRT files include:

- Subtitles appearing too early or too late in the video
- Overlapping cues that flash on screen too fast to read
- Merging subtitles from two sources into one file
- Splitting a long subtitle file into separate parts
- Converting to a different format like WebVTT

The good news: you can handle all of these without installing anything.

## How to Edit SRT File Timing Specifically

Timing is the most common SRT editing problem. Subtitles go out of sync when:

- The video has been re-encoded and the runtime changed
- The subtitle file was created for a different version of the video
- The subtitle file is offset from the very first frame

To fix timing in an SRT file online, use the [Subtitle Time Shifter](/subtitle-time-shifter). Enter a positive value (e.g. +2000ms) to push subtitles later, or a negative value (e.g. -1500ms) to pull them earlier. The tool updates every single timestamp in the file at once — no manual editing required.

## Common SRT Editing Mistakes to Avoid

**Manually editing timestamps in a text editor** — Shifting timestamps by hand is the slowest and most error-prone method. Use a dedicated time-shift tool instead.

**Ignoring overlapping cues** — Overlapping subtitles are one of the most common accessibility issues with subtitle files. Always run your file through an overlap checker before publishing.

**Saving in the wrong encoding** — SRT files should be saved in UTF-8 encoding. Most online editors handle this automatically, but desktop text editors sometimes default to other encodings that break special characters.

**Forgetting to validate** — After any edit, re-watch the video with the new subtitle file to spot-check sync at the beginning, middle, and end.
